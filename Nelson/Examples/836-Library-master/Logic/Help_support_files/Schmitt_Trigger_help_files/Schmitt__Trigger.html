Schmitt Trigger

VI Description: This is a code implementation of the electronic circuit behavior invented by Otto Schmitt in 1938. It provides a logical True OUT when the input IN exceeds the value for HI, but does not then return a logical False until IN falls below the value for LO. Because of this behavior, it acts as a threshold detector with hysterisis (which just means it acts differently depending on which direction the input is changing). It can be used to threshold an Analog input, insensitive to noise, as long as the noise level isn't too great.

Inputs:

IN - The numeric signal being tested for threshold.

HI - The high threshold value, above which will result in a True OUT.

LO -The low threshold value, below which will result in a False OUT.

Outputs:

OUT - Result of appliying IN to threshold test.

INV OUT - Inverse of OUT, representing an Inverting Schmitt-Trigger.

Implementation:

Determine values for HI and LO to create a seperation larger than the random variation (noise) present in IN.

Execution Properties:

Debugging is disabled for performance improvement.

Reentrancy is set to Preallocated Clone, which allows the VI to be used in multiple places, without confusing indivual VI's State.
